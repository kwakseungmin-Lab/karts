01 — System Overview

시스템 구조

karts 레포지토리는 단편영화 "명예의 무게"를 AI로 자동 제작하는 파이프라인입니다. Claude Code Workflow가 오케스트레이터로 동작하고 6개의 전문 에이전트가 각 단계를 담당합니다.
🎬

오케스트레이터

Claude Code Workflow
.claude/workflows/film-pipeline.js
JavaScript로 에이전트 실행 순서·병렬화·데이터 전달을 제어합니다.

🤖

전문 에이전트 6종

.claude/agents/에 마크다운으로 정의된 각 에이전트가 독립적 역할을 수행하며 출력물을 다음 에이전트에게 연결합니다.

🔧

도구 레이어

tools/ 패키지에 GPT Image, Sora, Higgsfield, FFmpeg 등 외부 API와 로컬 툴이 BaseTool 추상 계약으로 통합됩니다.

파일 구조

karts/ ├── .claude/ # Claude Code 설정 │ ├── agents/ # 에이전트 시스템 프롬프트 (마크다운) │ │ ├── storyboard-artist.md │ │ ├── frame-artist.md │ │ ├── img2video.md # Cinematographer │ │ ├── color-grader.md │ │ ├── video-editor.md │ │ └── film-critic.md │ └── workflows/ │ └── film-pipeline.js # 메인 오케스트레이터 스크립트 │ ├── tools/ # 도구 레이어 │ ├── base_tool.py # 모든 툴의 추상 베이스 클래스 │ ├── tool_registry.py # 자동 발견 레지스트리 │ ├── cost_tracker.py # 예산 관리 시스템 │ ├── video/ # Sora, Higgsfield, FFmpeg 등 34종 │ ├── graphics/ # GPT Image, FLUX 등 14종 │ ├── audio/ # TTS, 음악 생성 12종 │ └── enhancement/ # 색보정 │ └── short-film-project/ ├── plan/ # film_plan.md, color_grades.json ├── images/ # 캐릭터 시트, 컨셉아트, 배경 ├── storyboard/ # 생성된 스토리보드 JSON └── final/ ├── frames/ # 컷 이미지 PNG ├── videos/ # Sora 생성 원본 MP4 └── graded/ # 색보정 완료 MP4
02 — Pipeline Flow

파이프라인 전체 흐름

기획안(film_plan.md)과 컨셉아트를 시작점으로 6단계를 거쳐 완성 영상이 출력됩니다. Phase 1~4는 병렬 실행, Phase 5~6은 순차 실행입니다.
1
Storyboard
스토리보드 생성
⚡ parallel(12씬)
기획안과 컨셉아트를 읽어 12개 씬의 스토리보드 JSON을 생성합니다. 각 씬이 독립적으로 병렬 처리되며, Structured Output으로 스키마를 강제합니다.

병렬 처리 전략

Phase 1~4는 각 씬/컷이 독립적이므로 parallel()로 최대 병렬화합니다. Claude Code Workflow의 동시 에이전트는 min(16, cpu-2)로 캡 되고, 초과 작업은 큐에서 대기합니다.

Structured Output 활용

Phase 1에서 schema: STORYBOARD_SCHEMA 옵션으로 에이전트 출력을 강제 검증합니다. 워크플로우가 반환값에서 validBoards.flatMap(sb => sb.cuts)를 통해 타입 안전하게 컷 목록을 추출해 Phase 2에 투입합니다.

Higgsfield CLI 처리 흐름
# 1. 이미지 업로드 → upload_id 획득
upload_id=$(higgsfield upload create scene04.png --json \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['id'])")

# 2. 영상 생성 Job 제출 → job_id 획득
job_id=$(higgsfield generate create veo3_1_lite \
  --prompt "..." --image $upload_id --json \
  | python3 -c "import sys,json; print(json.load(sys.stdin)[0])")

# 3. 완료 대기 (최대 15분)
higgsfield generate wait $job_id --timeout 15m --json --quiet

FFmpeg filter_complex 구조

# 입력: [0]영상  [1]BGM  [2~11]SFX 10개

# BGM: 219초 트림 → 볼륨 0.45 → 마지막 5초 페이드아웃
[1:a]atrim=0:219,asetpts=PTS-STARTPTS,volume=0.45,
     afade=t=out:st=214:d=5[bgm]

# 각 SFX: ms 단위 adelay 배치 + 개별 볼륨
[2:a]adelay=54100|54100,volume=0.8[sfx0]  # 갑옷 발소리
[3:a]adelay=68000|68000,volume=0.9[sfx1]  # 검 발도
[4:a]adelay=85000|85000,volume=1.0[sfx2]  # 검 충돌
... (총 10개)

# 전체 합산 믹스
[bgm][sfx0][sfx1]...[sfx9]amix=inputs=11:duration=first:
     dropout_transition=2[aout]

# 영상은 복사, 오디오만 인코딩
-map 0:v -map [aout] -c:v copy -c:a aac -b:a 192k
BGM 볼륨 0.45 (영상 소리 묻히지 않게) SFX volume 0.6~1.0 (씬 긴장도 반영) -c:v copy → 영상 무손실 합성 출력: the_weight_of_honor_v4.mp4

Silent Runtime Swap 금지 원칙: video_compose.py에 "Silent runtime swaps are forbidden by governance"가 명시됩니다. 런타임(FFmpeg/Remotion/HyperFrames)은 proposal 단계에 고정되며, 실패 시 자동 대체 없이 structured blocker를 반환해 에이전트에게 재확인을 요청합니다.
⚠️
코드 내 유료 API 잔존: .env.example에서 FAL, Runway, Higgsfield, ElevenLabs, Suno API 키가 모두 주석("제거됨")으로 남아 있습니다. 하지만 sora_video.py, higgsfield_video.py 등 구현 코드에는 여전히 유료 API 코드가 존재합니다. 실제 실행 시 OPENAI_API_KEY가 없으면 Phase 3이 차단됩니다. 무료 대안: tools/video/ltx_video_local.py, wan_video.py.
